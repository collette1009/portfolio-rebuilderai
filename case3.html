반복 입력 자동화와 기준값 표준화를 통한 운영 데이터 정합성 확보

반복 입력 구간을 단축키 기반 자동화로 전환하되, 귀책 기준값 오입력을 줄이고 최종 확인은 사람에게 남겨 처리 속도를 높이고 데이터 정합성을 함께 확보한 사례입니다. 생성형 AI를 실무 파트너로 삼아 비개발자 환경에서 직접 로직을 설계·구현했고, 조직 내 AI 실무 활용의 첫 공식 승인 사례가 됐습니다.
#Automation #GenerativeAI #DataIntegrity

반복 입력이 속도만이 아닌 데이터 품질 문제였습니다

대량 취소·반품·재배송 처리 과정에서 동일한 항목을 반복 입력해야 하는 구간이 많아 처리 피로도와 입력 실수 가능성이 높았습니다.
미출고, 출고누락 등 케이스별로 귀책 대상이 달라지는데 담당자들이 자주 헷갈리면서 분류 오류가 반복되는 문제가 있었습니다.
물류 외부 운영 구조에서는 귀책 대상이 잘못 입력되면 원인 분석, 비용 정산, 운영 집계 데이터까지 함께 꼬일 수 있어 단순 속도보다 입력 기준의 표준화와 데이터 정합성 확보가 더 중요했습니다.

자동화 전에 기준을 먼저 만들었습니다

1. 입력 기준 먼저 정의
자동화 이전에 표준화
자동화를 하기 전에, 각 항목별로 어떤 값이 들어가야 하는지 기준을 먼저 정리했습니다. 재배송·환불·보상별 입력 항목 정의, 조건별 입력값 기준, 처리 유형에 따른 입력 흐름을 구분했습니다.
2. 자동화 대상 영역 선별
전체가 아닌 반복 가능한 영역만
업무 전체를 자동화하는 것이 아니라, 반복 입력과 귀책 기준값 자동 반영이 가능한 영역만 선별했습니다. 우리 귀책(재고 오산, 수급 문제 등)과 물류사 귀책(미출고, 포장 불량 등)을 케이스 유형에 따라 자동으로 입력되도록 설계해, 수동 분류에서 비롯되던 오류를 구조적으로 차단했습니다.
3. 자동화 로직 설계 및 적용
생성형 AI를 실무 파트너로
Chrome Extension, VBA, Apps Script로 반복 업무를 줄이는 보조자동화 도구를 구현했습니다. 생성형 AI를 로직 작성·수정·디버깅 보조에 활용해 비개발자 환경에서도 실제 업무 적용 수준까지 개선했습니다.
4. 수동 검수 구조 설계
사람이 꼭 해야 할 판단은 남겨둠
모든 과정을 자동화하지 않고, 최종 판단은 담당자가 확인하도록 남겨 데이터 예외 상황과 오입력 리스크를 통제했습니다.

어떤 성과를 얻었나요?

기존 8단계 반복 입력을 2단계로 축소
케이스 유형에 따른 귀책 기준값 자동 입력으로 수동 분류 오류 구조적 차단 — 잘못된 귀책 입력이 비용 정산·운영 집계를 꼬이게 하는 리스크 제거
정산 오류 및 보상 기준 혼선 감소 → 후속 처리 정확도 향상
생성형 AI 실무 활용 기여도를 조직에서 인정받아, 비개발 직군 중 유일하게 관련 도구를 업무용 비용으로 공식 승인받음

근거

1. 재배송 등록 자동입력 설계
재배송 등록 시 반복 입력하던 주요 항목 6개를 Alt + Shift + D 단축키 한 번으로 자동 입력되도록 설계했습니다. 귀책대상 등 케이스별 기준값을 함께 반영해 분류 오류를 줄였고, 마지막 등록은 직접 확인 후 진행하도록 남겼습니다.
2. 반품 등록 자동입력 설계
반품 등록 시 반복 입력하던 주요 항목 6개를 Alt + Shift + S 단축키로 자동 입력되도록 구성했습니다. 반복 입력 시간을 줄이는 동시에 기준값 오입력을 줄여 데이터 정합성을 높였습니다.
3. 생성형 AI 실무 도구화 및 조직 승인
비개발자 환경에서도 생성형 AI를 로직 작성·수정·디버깅 보조에 실무적으로 활용했고, 그 기여도를 인정받아 비개발 직군 중 유일하게 관련 도구를 업무용 비용으로 전환했습니다.

배운 점

1
운영 자동화의 핵심은 단순히 빠르게 처리하는 것이 아니라, 반복적으로 발생하는 분류 실수를 구조적으로 막는 것이었습니다.
2
좋은 자동화는 모든 단계를 대신하는 것이 아니라, 반복 입력은 줄이고 최종 확인은 사람에게 남겨 리스크를 통제하는 구조가 운영 안정성을 높이는 데 중요했습니다.
3
비개발자도 생성형 AI를 활용하면 로직 설계와 도구 구현이 가능합니다. 도구의 한계가 아니라 어떻게 활용하는지가 핵심이었습니다.
